# 局域网 P2P 即时通信与文件传输系统 —— 安装与运行部署说明书

## 一、 环境前置依赖准备
在部署和运行本项目之前，请确保测试宿主机已配置好以下标准开发运行环境：
1. **Node.js 运行时**：建议版本为 `v18.x` 或 `v20.x` 及以上的 LTS 稳定版（项目底层深度依赖 Node 原生底层网卡套接字及高性能 Stream 二进制流）。
2. **包管理工具**：系统原生支持 `npm`，亦可使用速度更快的 `yarn` 或 `pnpm`。
3. **IDE 编辑器推荐**：Visual Studio Code（建议安装 Volar 插件以获得最佳 Vue 3 强类型安全语法高亮）。

## 二、 依赖本地安装
1. 打开 Windows PowerShell（建议以管理员身份运行）或终端，切换到解压后的项目根目录下：
  ```powershell
  # 提示：请根据你本地实际存放该项目的路径进行切换
  cd <你的项目所在本地路径>
  ```

2. 执行以下命令一键安装前端构建工具（Vite）、桌面运行时（Electron）以及 Vue 核心依赖树：
  ```powershell
  npm install
  ```

## 三、 单机回环多开测试指南 (答辩演示核心步骤)

为了在一台测试机上完美模拟分布式 P2P Mesh 局域网的上线盲搜和套接字对等通信，本项目设计了**自适应端口漂移与缓存锁环境隔离机制**。请严格按照以下步骤开启两个对等方窗口：

补充说明：双方互传文件后，接收到的文件会自动保存到项目根目录下的实例专属文件夹中。第一个实例默认写入 `received_files_0`，第二个实例默认写入 `received_files_1`。这两个目录属于运行期产物，已经建议加入 `.gitignore`，不要上传到 GitHub。

### 1. 启动第一个 P2P 节点实例（默认主节点）

在第一个终端中，执行：

```powershell
npm run dev
```

* **挂载状态**：主窗口前端在本地 `http://localhost:5173/` 拉起。主进程全自动激活底层网络监听：UDP 盲搜绑定在 `41234` 端口，TCP 消息接收挂载在 `41240` 端口，TCP 文件接收挂载在 `41300` 端口。

### 2. 启动第二个 P2P 节点实例（对端节点）

**新建一个独立的终端窗口**，同样切换到项目根目录下，执行：

```powershell
npm run dev:2
```

* **防碰撞隔离状态**：该命令会强行注入 `cross-env PORT_OFFSET=1` 偏移动态环境。第二个窗口全自动防碰撞漂移：前端代理在 `5174` 端口，UDP 盲搜目标靶向调转，TCP 文本接收自增漂移至 `41250` 端口，TCP 文件接收自增漂移至 `41310` 端口。同时自动将 `userData` 缓存路径隔离，避开了 Windows 系统级的锁冲突拒绝访问报错。

### 3. 文件传输与保存位置

1. 在聊天面板点击加号按钮后，可选择任意本地文件发送给当前对端。
2. 接收成功后，文件会自动落盘到当前项目根目录下的 `received_files_0` 或 `received_files_1` 目录，具体取决于当前实例的 `PORT_OFFSET`。
3. 如果传输的是 `.png`、`.jpg`、`.jpeg`、`.webp`、`.gif`、`.bmp`、`.svg` 等图片文件，聊天气泡中会直接显示预览图，便于确认收发结果。

## 四、 真实多物理机局域网部署

若需要将系统移至连着同一个 Wi-Fi 或交换机的两台真实电脑进行物理互传调测，部署流程如下：

1. 分别在两台物理机上拷贝源码并执行 `npm install`。
2. **两台电脑同时直接执行** `npm run dev` 即可（无需执行 `dev:2`，因为在物理隔离环境下，它们各自运行在默认端口，不会发生本机端口冲突）。
3. 两台物理机通过底层的 UDP 有限局域网广播自动盲搜寻，界面上展现的对端网络标牌会自动由 `127.0.0.1` 自适应转换为对方真实的物理局域网网卡 IP（如 `192.168.x.x`），实现真正的分布式 P2P 直连。

## 五、 典型故障排错指南

1. **报错 `EADDRINUSE` (端口已被抢占)**：
* 原因：之前运行的 Electron 进程残留，未完全在后台杀死，导致独占了套接字。
* 解决办法：关闭终端，或者在 Windows 任务管理器中强行结束所有名为 `Electron` 的后台进程，随后重新键入指令拉起。


2. **前端界面修改后未实时刷新 (HMR 挂起)**：
* 解决办法：项目完整集成了 Vite 8 的模块热替换。若突发挂起，在渲染视窗中按下 `Ctrl + R`（或在顶部菜单选择 `View -> Reload`）即可强制无损重载前端渲染树，无需重启整个主进程。
